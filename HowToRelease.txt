Memorandum of How To Release.

1. update change logs.
  - update product.xml in changelog/product.xml.
  - regenerate ChangeLog.html:
    % mvn generate-resources

2. update version number.

  - pom.xml
    <version>X.Y.Z</version>

  - src/main/resources/samurai-graph.properties
    (version is automatically filtered from pom.xml)

3. commit version updated sources.
  % git commit

4. add tag to GIT repository.
  % git tag vX.Y.Z

5. rebuild binaries.
  % mvn clean package

6. create os specific installer (using jpackage)

   The jpackage tool requires building on the target platform.
   Windows installers must be built on Windows, macOS packages on macOS,
   and Linux packages on their respective distributions.

   Windows (run on Windows):
     mvn clean package -Pjpackage-windows
     Output: target/dist/Samurai Graph-<version>.exe

   macOS (run on macOS):
     mvn clean package -Pjpackage-mac
     Output: target/dist/Samurai Graph-<version>.dmg

   Linux (run on Linux):
     JAVA_HOME=/path/to/temurin-jdk mvn clean package -Pjpackage-linux
     Output: target/dist/samurai-graph-<version>.deb
             target/dist/samurai-graph-<version>.rpm

   Note: On Fedora/RHEL, the system OpenJDK package modifies java.security,
   which causes jlink (used by jpackage) to fail with:
     "Error: .../java.security has been modified"
   Workaround: use a non-distro JDK (e.g. Eclipse Temurin) via JAVA_HOME:
     JAVA_HOME=/usr/lib/jvm/temurin-21-jdk mvn clean package -Pjpackage-linux
